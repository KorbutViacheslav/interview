Types of Classes:
    -Final Class
    -Static Class
    -Abstract Class
    -Concrete Class
    -POJO Class
    -Singleton Class
    -Inner Class

-Final Class:
    Клас, який не може бути розширений (унаслідуваний).
Це означає, що від нього не можна створити підкласи.
Використовується для захисту класу від змін та оптимізації коду.

-Static Class:
    Клас, який не може бути інстанційований (створений об'єкт).
Він містить лише статичні методи та змінні, які належать до самого класу, а не до його об'єктів.

-Abstract Class:
    Клас, який не може бути інстанційований, але може мати абстрактні методи (методи без тіла).
Абстрактні класи використовуються для визначення загального інтерфейсу для підкласів.

-Concrete Class:
    Протилежність абстрактного класу. Це звичайний клас,
який може бути інстанційований і всі його методи мають реалізацію.

-POJO Class:
    Plain Old Java Object. Простий клас Java,
який не дотримується жодних специфічних умов або інтерфейсів.
Зазвичай використовується для представлення даних.

-Singleton Class:
    Гарантує, що в будь-який момент часу існує лише один екземпляр цього класу.
Використовується для управління ресурсами, кешування даних та інших завдань,
де потрібен єдиний об'єкт.

-Inner Class:
    Клас, який визначений всередині іншого класу.
Може бути статичним або нестатичним, локальним або анонімним.
Використовується для організації коду та створення більш специфічних класів.
